# Weather Dashboard

## Краткое описание

Weather Dashboard — это учебное Android-приложение, демонстрирующее работу корутин в Kotlin. Оно имитирует загрузку данных о погоде из трёх источников (температура, влажность, скорость ветра) с возможностью параллельной загрузки, обработкой ошибок и автоматическим обновлением.

## Функциональность приложения

- Параллельная загрузка погодных данных с помощью `async` (общее время ~2 секунды)
- Отображение индикаторов загрузки для каждого параметра
- Кнопка ручного обновления (`Refresh Weather`)
- Имитация ошибки сети с кнопкой `Simulate Error` и корректной обработкой исключений
- Расчёт «индекса погоды» на основе тяжёлых вычислений с использованием `Dispatchers.Default`
- Автоматическое обновление данных каждые 10 секунд
- Отображение текстового прогресса загрузки
- Адаптивный интерфейс на Jetpack Compose

## Технологии и библиотеки

- **Kotlin** — основной язык разработки
- **Jetpack Compose** — декларативный UI
- **Coroutines** (`kotlinx.coroutines`) — асинхронные операции
- **ViewModel** (`lifecycle-viewmodel-compose`) — управление состоянием
- **StateFlow** — реактивные потоки данных
- **Material 3** — компоненты интерфейса

## Контрольные вопросы

### A) В чём разница между `launch` и `async`?

- `launch` запускает корутину и не возвращает результат (возвращает `Job`). Используется для задач «запустил и забыл» (логирование, отправка аналитики).
- `async` запускает корутину и возвращает `Deferred<T>`, из которого можно получить результат через `await()`. Используется, когда нужен возвращаемый результат (например, загрузка данных).

**Пример кода:**
```kotlin
// launch — без результата
viewModelScope.launch {
    sendAnalyticsEvent()
}

// async — с результатом
val deferred = viewModelScope.async {
    fetchUserData()
}
val userData = deferred.await()
```

### B) Что такое suspend функция?

`suspend` — это функция, которая может приостановить своё выполнение без блокировки потока. Она может вызываться только из другой `suspend`-функции или из корутины.

**Может ли она вызываться из обычной функции?**  
Нет, напрямую нельзя. Для вызова нужен корутинный контекст (например, `launch` или `runBlocking`).

**Почему `delay()` не блокирует поток?**  
`delay()` приостанавливает только корутину, а поток продолжает выполнять другие задачи. В отличие от `Thread.sleep()`, который блокирует весь поток.

### C) Зачем нужны разные диспетчеры?

Диспетчеры определяют, на каких потоках выполняется корутина.

| Диспетчер | Когда использовать | Пример задачи |
|-----------|--------------------|----------------|
| `Dispatchers.Main` | Обновление UI | Изменение текста в `TextView` |
| `Dispatchers.IO` | Сеть, файлы, БД | Загрузка картинки из интернета |
| `Dispatchers.Default` | Тяжёлые вычисления | Обработка большого списка |

**Что будет, если выполнить тяжёлое вычисление на `Dispatchers.Main`?**  
UI-поток заблокируется, приложение перестанет отвечать на нажатия (ANR — Application Not Responding).

### D) Что произойдёт, если не обработать исключение в корутине?

Необработанное исключение в корутине, запущенной через `launch` или `async`, приведёт к крашу приложения (если не установлен `CoroutineExceptionHandler`). Исключение всплывает в родительскую корутину и отменяет её.

**Как корректно обрабатывать ошибки?**
- Использовать `try-catch` внутри корутины или в блоке `async`/`launch`.
- Оборачивать группу корутин в `coroutineScope` — тогда любая ошибка отменит все дочерние задачи и попадёт в `catch`.
- Использовать `SupervisorJob` для изоляции ошибок.

**Зачем нужен `try-catch` внутри `launch`?**  
Чтобы перехватить исключения, возникшие в корутине, и обработать их (например, показать сообщение пользователю) без краша приложения.

### E) Как работает автоматическая отмена корутин?

Корутины отменяются автоматически при отмене их `Job` или при завершении области видимости (`CoroutineScope`).

**Что такое `viewModelScope`?**  
Это встроенная область видимости для `ViewModel`. Она привязана к жизненному циклу `ViewModel` и автоматически отменяет все запущенные корутины, когда `ViewModel` уничтожается (например, при закрытии экрана).

**Когда корутины отменяются автоматически?**
- При вызове `ViewModel.onCleared()`
- При завершении `lifecycleScope` (для Activity/Fragment)
- При явном вызове `cancel()` на `Job`
- При отмене родительской корутины

## Как запустить проект

1. Клонируйте репозиторий и откройте проект в Android Studio.
2. Синхронизируйте проект (нажмите `Sync Now` в `build.gradle.kts`).
3. Подключите эмулятор Android или физическое устройство и нажмите `Run`.
